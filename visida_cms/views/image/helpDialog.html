Help
I can't add or edit items.

If the record is marked as Completed, you will not be able to add or edit new items. If an item is wrong and needs review, make a request for an admin.
If the record is not completed, you may be restricted in what you can modify. An administrator can assign you to a study with identification only, quantification only, both together or view only permissions. Please contact an administrator if you believe your permission level to be incorrect.

This record is invalid/incorrect.

If this record is a duplicate, has an unusable image or is invalid in some other way it should be hidden from the study so it will not pollute results.
This can be done by clicking the (eye) symbol so it becomes . This will continue to display the record to you and other administrators, but will hide it from any analysts and will not include any data from this record in any system output. This can only be done by an admin. If you believe this is required, make an admin request.

Participant or guest metadata is incorrect.

If you believe the listed participants or guest count to be inaccurate, this can be corrected.
To show or hide an original participant, click the symbol adjacent to the participant so it becomes (or vice-versa to hide). To add a missing participant, use the "Add Participant.." option under the participants. Hidden participants will continue to display to you and other administrators, but will hide it from any analysts and will not be included in any system output. This can only be done by an admin. If you believe this is required, make an admin request.

Making an admin request.

Admin requests made by other users will appear on your Home screen as a comment tagged with the symbol. You can also make an admin request, for example to seek advice about a difficult item, if there is other admin accounts. This is done by making a new comment with the symbol turned on.

Some tasks such as removing/editing completed records or metadata are retricted to admin accounts. If you believe data in a record to be incorrect, alerting an admin can allow them to repair it. Administrators can also assist with identifying food items or difficult records. To alert an admin, add a new comment with the symbol turned on or use a following option:

There is an issue with this item.

Clicking an option below will create a comment and alert an coordinator to your issue. You may be asked for more details.

  • I'm unsure about the best match for this item in the database.
    I can identify the item, but there is no exact match in the food list.
  • I cannot identify this item.
    The information available in the image, voice recording and/or text description is not enough to code.
  • I cannot quantify this item.
    There is not a suitable tool available to quantify the item.
  • This record is a duplicate.
    This exact record appears more than once.
  • There is an issue with the Metadata of this record.
    Click to expand. There is an issue with record type, participants or plate type.
    • The record type is incorrect.
      The record type listed in the metadata is incorrect. For example, the record is captured as an eating occasion, but is actually an ingredient for a recipe.
    • There is an error with the participant(s) for this record.
      The record type listed in the metadata is incorrect. For example, the record is captured as an eating occasion, but is actually an ingredient for a recipe.
    • There is an error with the total number of individuals eating.
      The total number of individuals eating the shared dish listed in the metadata is incorrect.
  • This record is not valid.
    This record is not an eating occasion or recipe.
  • This record contains identifying information.
    This record, in addition to food items, contains information that may identify a person.
  • I have another issue.
    My issue is none of the above options.
